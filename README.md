# M-Waves (الموجة التاريخية) — مؤشّر TradingView Pine Script v6

استراتيجية تداول تجمع موجات إليوت + نظرية داو + مفاهيم السيولة الذكية (SMC)، منفّذة كمؤشّر
**Pine Script v6**. هذا الفرع (`rebuild-from-scratch`) لإعادة البناء النظيف طبقةً طبقةً.

## هيكل المشروع

```
.
├── J-Waves-M1.pine            ← الإصدار الجديد قيد البناء (النواة النظيفة)
├── M-Waves_M1.pine            ← الكود القديم (مصدر سحب الأجزاء المثبَتة فقط)
├── skills-lock.json
├── docs/
│   ├── build/                 ← الوثائق الحاكمة لإعادة البناء (تُتَّبع حرفيًّا)
│   │   ├── MWaves_coding_charter.md      دستور كتابة الكود (الأعلى سلطةً)
│   │   ├── MWaves_architecture_full.md   المعمارية الكاملة (الطبقات ط0→ط4)
│   │   ├── MWaves_buildplan.md           خطة البناء (12 مرحلة)
│   │   └── MWaves_inventory.md           جرد ما نُبقي/نحذف/نؤجّل
│   ├── strategy/             ← حقيقة الاستراتيجية (منطق الدروس + قائمة التحقّق + الكتب)
│   └── archive/              ← تقارير/خطط/أدلّة قديمة (تاريخ، ليست مرجعًا حاكمًا)
├── reference/                ← كود مرجعيّ خارجيّ للمقارنة (لا يُنسخ منه منطق درس)
└── .claude/skills/
    ├── mwaves-rebuild/       ← دستور إعادة البناء (يُقرأ أوّل كل عمل تطوير)
    └── mwaves-engineer/      ← قناع الخبراء الستّة (للكود القديم)
```

## كيف نعمل

1. اقرأ `docs/build/` بالترتيب (الدستور → المعمارية → الخطة → الجرد) قبل أيّ كود.
2. معمارية فراكتالية أحادية الاتجاه: ط0 الأساس → ط1 القمم/القيعان → ط2 الخطوط → ط3 الاستنتاجات/المراحل → ط4 المناطق/الفرص. لا طبقة تقرأ من فوقها.
3. **تغيير واحد قابل للاختبار** لكل دورة، يُطابَق التحليل اليدويّ على شارت حقيقيّ قبل التالي.
4. القاعدة الجوهرية كور بلا input؛ العتبات نِسَب من سعة الموجة (لا مؤشّر أجنبيّ، لا رقم سحريّ).

> **الروح:** نطبّق الاستراتيجية بالضبط، وما لا يمكن أتمتته نفكّر بطريقة أمينة له ونوثّق الفرق.
