# M-Waves — دستور إعادة البناء النظيف

أنت تعمل على إعادة بناء مؤشّر **M-Waves (الموجة التاريخية)** من الصفر (فرع `rebuild-from-scratch`)،
استراتيجية تداول عربية تجمع موجات إليوت ونظرية داو ومفاهيم السيولة الذكية (SMC)، منفّذة كمؤشّر
TradingView **Pine Script v6**. كل التواصل والتوثيق **بالعربية**.

## ١) اقرأ الوثائق الحاكمة أوّلًا — لا كود قبلها

قبل أيّ تحليل أو سطر كود، اقرأ من `docs/build/` (مصدر الحقيقة، لا تعتمد ذاكرتك):

1. **`docs/build/MWaves_coding_charter.md`** — دستور كتابة الكود (الأعلى سلطةً). مبادئ المعماريّ +
   المطوّر + طقوس كل دورة + **الخطوط الحمراء**. عند تعارض «أسرع» مع «أنظف» ⇒ الأنظف يفوز.
2. **`docs/build/MWaves_architecture_full.md`** — الطبقات ط0→ط4، الفلسفة الفراكتالية، عقود الطبقات،
   أنواع البيانات (`Proc`/`DegMem`/`Phase`/`LiqBreak`).
3. **`docs/build/MWaves_buildplan.md`** — التسلسل (12 مرحلة): لكل مرحلة نطاق/نبني/نستبعد/معيار
   قبول/يعتمد على + بوّابات الجودة.
4. **`docs/build/MWaves_inventory.md`** — ما نُبقي/نحذف/نؤجّل لكل درس.

لِحقيقة الاستراتيجية (تعريفات الدروس الدقيقة) ارجع إلى `docs/strategy/` — خصوصًا
`M-Waves_منطق_الدروس.md` و `M-Waves_قائمة_التحقق.md`. هذه الاستراتيجية لها تعريفاتها الخاصّة؛ لا
تستعمل فهمك العامّ لـ«إليوت/داو».

## ٢) الخطوط الحمراء (لا تُعبَر أبدًا)

🔴 مؤشّر أجنبيّ (ATR/RSI/MACD/LuxAlgo) في قاعدة جوهرية — العتبات **نِسَب من سعة الموجة** فقط.
🔴 **دائرة:** طبقة تقرأ من أعلى. كل طبقة تقرأ ممّا **تحتها** فقط (صغيرة→متوسطة→كبيرة).
🔴 مسار مزدوج لنفس الوظيفة · رقم سحريّ مخترَع · حالة تعبر حدّها (الموجة).
🔴 تكديس فوق القديم بدل استبداله · كتابة كود قبل اعتماد المواصفة.
🔴 تجاوز المرحلة الحالية قبل اختبارها ومطابقتها يدويًّا · التخمين بدل السؤال عند الغموض.

عند ملامسة أيّ خطّ أحمر ⇒ **توقّف واطرح السؤال**، لا تكتب ولا تخمّن (Fail Loud).

## ٣) المعمارية الفراكتالية (الاتجاه — كل سهم للأعلى)

```
ط4 المناطق/الفرص [دروس 01-03]  ← الأعلى
ط3 الاستنتاجات/المراحل [10-11]
ط2 الخطوط/التجدّد [08-09]
ط1 القمم/القيعان [05-07]  (صغيرة→متوسطة→كبيرة)
ط0 الأساس: leg + الموجة التاريخية [04]  ← الأسفل
```
ترتيب التنفيذ داخل كل بار: صغيرة → متوسطة → كبيرة → خطوط → استنتاجات/مراحل → مناطق/فرص.
المحرّك الموحّد: نفس آلية الكسر (`Proc`/feed) تشتقّ كل درجة من التي تحتها. قواعد موحّدة عبر
الدرجات: **الفعالية (تغطية) · التتابُع · التفعيل**.

## ٤) طقوس كل دورة (إلزاميّة)

**قبل:** اقرأ الكتلة المعنيّة (جرد الموجود) → حدّد الطبقة/الدرس وتأكّد ألّا دائرة → اعتمِد المواصفة.
**أثناء:** تغيير **واحد** قابل للاختبار · **استبدال لا إضافة** (احذف ما عفا) · أسماء معبّرة
(`large/medium/small`، `legMinRatio` لا `lmr`) · تعليق «لماذا» يربط كل قاعدة بدرسها · لا رقم
سحريّ · Pine v6: **أمر واحد لكل سطر، بلا فاصلة منقوطة**.
**بعد:** اعرض diff واضحًا + سطرَي «المتوقّع بصريًّا على الشارت» → حدّث وثائق البناء/الفجوات →
**توقّف وانتظر** اختبار المستخدم ومطابقته اليدوية قبل التالي.

## ٥) معيار القبول وبيئة الاختبار

- المعيار دائمًا = **مطابقة التحليل اليدويّ على شارت حقيقيّ** (الذهب 1H غالبًا)، لا «الكود يبدو صحيحًا».
- **لا أستطيع رؤية TradingView** (لا MCP؛ المؤشّر يُرسَم بمتصفّح المستخدم). ⇒ المستخدم يلصق الكود،
  يختبر بصريًّا، ويرسل لقطات شاشة أقرؤها. المُصرِّف الوحيد = TradingView.
- القاعدة الجوهرية **كور بلا input**؛ الـ input للضبط البصريّ فقط (لون/نسبة معايرة/إظهار).

## ٦) المرجع والملفّات

- `docs/build/` — الوثائق الحاكمة الأربع (اقرأها أوّلًا، بهذا الترتيب).
- `docs/strategy/` — حقيقة الاستراتيجية (منطق الدروس + قائمة التحقّق + الكتب).
- `M-Waves_M1.pine` — الكود القديم (مصدر **سحب الأجزاء المثبَتة فقط**، لا النسخ الأعمى).
- `reference/` — كود مرجعيّ خارجيّ للمقارنة (لا يُنسخ منه منطق درس).
- `docs/archive/` — تقارير/خطط/أدلّة قديمة (تاريخ، ليست مرجعًا حاكمًا).

> **الروح:** «نطبّق الاستراتيجية بالضبط، وما لا يمكن أتمتته نفكّر بطريقة أمينة له ونوثّق الفرق.»
> الكود مرآة للاستراتيجية: بسيط بقدرها، نظيف بأمانتها، فراكتاليّ ببنيتها. كل سطر له مبرّر من درس.
