## 1) `useRenewal` — تجدّد خط السيولة (يخصّ [R2])

### ماذا يفعل (عند التفعيل)
- يستدعي `structuralBreak` على خط السيولة (المتوسطة 963‑964 والصغيرة 1008‑1009).
- عند تأكّد الكسر بشمعتين ⇒ **يمسح خط السيولة** (967‑969 / 1012‑1014).
- البوّابة `medGate`/`smGate` تُدخِل `brkBar` ⇒ **لا يُعاد رسم الخط إلا بنقطتين جديدتين** بعده (971 / 1015).

### دالّة الكسر `structuralBreak` (334‑349)
```pine
if dir == 1   // سيولة صاعدة (على القيعان)
    lb.cnt := (c < o and c < liqLvl) ? lb.cnt + 1 : 0   // شمعة حمراء تحت الخط
    if lb.cnt >= 2 ⇒ كسر مؤكّد
if dir == -1  // سيولة هابطة (على القمم)
    lb.cnt := (c > o and c > liqLvl) ? lb.cnt + 1 : 0   // شمعة خضراء فوق الخط
    if lb.cnt >= 2 ⇒ كسر مؤكّد
```

### المطابقة للدرس
| نصّ الدرس [R1]/[R2] | الحالة | الدليل |
|---|---|---|
| الكسر = **شمعتان** | ✅ مطابق | `lb.cnt >= 2` |
| صعوداً: **شمعتان حمراوان تحت** الخط | ✅ مطابق | `c < o and c < liqLvl` |
| هبوطاً: **شمعتان خضراوان فوق** الخط | ✅ مطابق | `c > o and c > liqLvl` |
| عند الكسر: **امسح** الخط | ✅ مطابق | `line.delete(memM.liq)` |
| **انتظر نقطتين جديدتين** ثم ارسم | ✅ مطابق | البوّابة تشترط نقطتين بعد `brkBar` |

**الحكم:** ✅ **منطق `useRenewal` مطابق للدرس 09 [R1]+[R2] حرفياً.**
**الانحراف الوحيد:** **مطفأ افتراضياً** (`false`) وموسوم «قيد الاختبار». بينما [R2] **قاعدة جوهرية** في درس التجدّد ⇒ يُفترض أن يكون **مفعّلاً افتراضياً** (بعد اختبار استقراره).

## 2) `useSmallTL` — كسر مستوى الاتجاه الصغير (يخصّ [S2]/[LIQ2])

### ماذا يفعل (عند التفعيل، 988‑1000)
- `smTL` = جسم آخر قاع/قمة صغيرة (مستوى الاتجاه الصغير).
- شمعتان مغلقتان خلفه ⇒ **يقلب `smallActiveDir`** ويضبط `smallWaveStart` (موجة صغيرة جديدة).

### المطابقة للدرس
| نصّ الدرس | الحالة | الدليل |
|---|---|---|
| كسر مستوى الاتجاه بشمعتين يؤكّد تغيّر الهيكل [LIQ2] | ✅ مطابق | `smTLcnt >= 2` |
| صعوداً: شمعتان حمراوان تحت آخر قاع صغير | ✅ مطابق | `close < open and close < smTL` |
| «بدء موجة صغيرة جديدة عند كسر آخر قاع/قمة» [S2] | ✅ مطابق | قلب `smallActiveDir` + `smallWaveStart` |

**الحكم:** ✅ **منطق `useSmallTL` مطابق للدرس [R1]+[LIQ2]+[S2].**

### ⚠️ ملاحظة تعايُش آليتين (تحتاج انتباهاً قبل التفعيل)
بعد إصلاح [A]، يُقلَب `smallActiveDir` افتراضياً ببنية الاتجاه (قمة أعلى/قاع أدنى عند انبثاق نقطة صغيرة، 753‑758). فإن فُعّل `useSmallTL`، تتعايش آليتا قلب:
- البنيوية (انبثاق نقطة — فورية، بلا اشتراط شمعتين).
- كسر المستوى بشمعتين (988‑1000 — مطابقة [R1]).
قد تتعارضان أو تُكرّران القلب. **لا أثر في الوضع الافتراضي (مطفأ)**، لكن **قبل تفعيله يجب التحقّق من عدم تضاربهما** (الأفضل: عند التفعيل، يكون كسر الشمعتين هو المُعتمَد، وتُلغى/تُكبَح الآلية البنيوية للقلب).

> تنويه دقّة: الدرس [R1] يشترط «شمعتين» لأي كسر. الآلية الافتراضية (قلب ببنية فورية) **أقلّ صرامة** من نصّ «الشمعتين»؛ و`useSmallTL` هو المسار الصارم (شمعتان) لكنه مطفأ. هذا فرق دقيق بين «الافتراضي العملي» و«حرفية الدرس».
