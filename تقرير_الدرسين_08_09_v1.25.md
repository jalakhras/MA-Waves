## أهمّ تغيير منذ v1.18: ✅ تكدّس الشموع عُولِج (قصّ ضمن نطاق الموجة)

السيولة ما زالت تُسقَط بالميل، **لكن الطرف المُسقَط يُقصّ الآن ضمن نطاق الموجة** `[hwLoBand, hwHiBand]` — في **الدرجات الثلاث**:

```pine
// الصغيرة/المتوسطة (updateDeg, 264‑279):
int rbU = last_bar_index + liqExt
if slU > 0 and not na(bandHi)
    rbU := math.min(rbU, math.max(m.lbBar, m.lbBar + int((bandHi - m.lbP) / slU)))   // لا يتجاوز قمة الموجة
if slU < 0 and not na(bandLo)
    rbU := math.min(rbU, math.max(m.lbBar, m.lbBar + int((bandLo - m.lbP) / slU)))   // لا يتجاوز قاع الموجة
// الكبيرة (1498‑1516): نفس القصّ على rbBu/rbBd ضمن hwHiBand/hwLoBand
```
والنطاق يُحسب من الطول التلقائي لا اليدوي (605‑611) لتفادي تضخّمه:
```pine
int bandLen = math.min(math.max(autoLen, 1), bar_index + 1)
float hwHiBand = ta.highest(high, bandLen)
float hwLoBand = ta.lowest(low, bandLen)
```

**التقييم:** ✅ **صحيح وحلٌّ مشتقّ لا تعسّفي.** بحكم القصّ، السعر المُسقَط **لا يتجاوز نطاق الموجة (= نطاق الشموع)** ⇒ لا يتمدّد المقياس ⇒ **لا تنضغط الشموع.** والخط يبقى يمرّ بالنقطتين ويمتدّ بقدر ما يسمح النطاق — فالوصل [LIQ1] محفوظ. **بند التكدّس المؤجَّل أُغلق.**

> يلزم تأكيدك البصري النهائي: تشغيل المؤشر دون انضغاط الشموع (خاصة مع موجة حادّة الميل أو تحديد يدوي بعدد شموع كبير).

## تعديل مُضاف (بطلب المستخدم) — قانونان إضافيان

**قانون [NEW‑1] — إنشاء القمم/القيعان الزرقاء (الكبيرة) بالكسر:**
الكبيرة يجب أن تُنشأ بنفس قانون المتوسطة الصفراء، لكن على درجة أعلى: المتوسطة تُشتق بكسر قمة/قاع **صغير**؛ والكبيرة تُشتق **بكسر قمة/قاع متوسط** (نفس آلية Proc/feed): صعوداً عند كسر قاع متوسط ⇒ أعلى سعر قبله = قمة زرقاء، أدنى بعده = قاع أزرق؛ والعكس هبوطاً. **قانون صحيح لم يُذكر في الدروس** (امتداد منطقي لقاعدة «كل درجة من التي تحتها بالكسر»). الكود الحالي يبني الكبيرة بـ«تجميع legs» تقديري (bigMinSwings/ATR/Bars) لا بالكسر النظيف ⇒ **يجب استبداله/تكميله بقانون الكسر.** (الدفعة 9.)

**قانون [NEW‑2] — تحديث الخطوط على كل نقطة جديدة (أي درجة):**
عند إنشاء أي قمة/قاع جديدة من **أي درجة** ⇒ يجب تحديث **خط السيولة** (آخر قاعين/قمتين) و**خط تحديد الاتجاه** (آخر قاع/قمة) لتلك الدرجة فوراً، باتجاه موجتها النشطة، مع القصّ ضمن النطاق (لا تكدّس) والبوّابة (آخر موجة فقط).
- الصغيرة والمتوسطة: ✅ مُطبّق (`updateDeg` يُستدعى على كل نقطة).
- الكبيرة: ⚠️ حالياً سيولتها/مستواها يُعاد حسابهما عند `islast` فقط؛ المطلوب استدعاء `updateDeg` لذاكرة الكبيرة **عند انبثاق كل نقطة زرقاء جديدة** (بعد تطبيق [NEW‑1])، بنفس نمط الصغيرة/المتوسطة.

> هذان القانونان أُدمجا في **الدفعة 2** (v1.20) من البرومت الشامل مع توحيد اتجاه الكبيرة [D] — لأنها جميعاً تخصّ بنية الكبيرة (تفادياً لعمل مزدوج).
