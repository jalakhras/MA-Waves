## أهمّ تغيير منذ v1.28: ✅ [D] توحيد اتجاه الكبيرة — تمّ

`bigDir` (النافذة المتحرّكة) لم يَعُد مصدراً منافساً، بل **احتياطاً أخيراً (bootstrap) فقط**. كل الدرجات تستند الآن إلى المصدر البنيوي أولاً:

```pine
// الصغيرة/المتوسطة صارت: المصدر البنيوي largeDir أولاً، ثم bigDir احتياط أخير فقط
int sBoot  = largeDir != 0 ? largeDir : bigDir                                  // 793
int sActive = smallActiveDir != 0 ? smallActiveDir : (largeDir != 0 ? largeDir : bigDir)  // 833
int corrDir = medActiveDir != 0 ? medActiveDir : (largeDir != 0 ? largeDir : bigDir)       // 890
int mActive = medActiveDir != 0 ? medActiveDir : (largeDir != 0 ? largeDir : bigDir)       // 930
```
- `bigDir` يبقى يُستعمل فقط عند `largeDir == 0` (أول الشموع — تمهيد مشروع). 
- **الازدواج البنيوي [D] انتهى عملياً:** الاتجاه الحاكم لكل الدرجات = البنية (`largeDir`/الاتجاه النشط)، لا النافذة.

**التقييم:** ✅ **[D] مُغلق** (كان تناقضاً بنيوياً مزمناً). 
> ملاحظة توثيق: تعليق الرأس (سطر 10) وسطر 940 ما زالا يقولان «اتجاه الكبيرة = bigDir» — **تعليق قديم يحتاج تحديثاً** (لم يَعُد دقيقاً).
